# 核心流程

本文整理診所開業時間、醫師上班時間、接診方式與病患端線上預約格之間的核心關係。

## 時格商品模型

系統每天都會提供 1440 個 1 分鐘時格。

```text
24 小時 x 60 分鐘 = 1440 格
```

每一個時格都是獨立單位，可以理解成一個商品。時間是這個商品的品名，例如：

```text
2026-05-28 09:00
2026-05-28 09:01
2026-05-28 09:02
```

時格自系統建立後就不會消失。後續所有操作都不是新增或刪除時格，而是記錄這個時格被哪些診所、哪些醫師、哪些使用者使用，以及使用狀態。

### 系統層

系統是時格工廠，每天提供 1440 個基礎時格。

系統層時格永遠存在，不因診所停業、醫師停診、暫停預約或預約取消而消失。

### 診所層

診所可以理解成大盤商。

診所營業時間不是創造時格，而是從每天 1440 個時格中選擇哪些時格要使用。時段選擇器只是批量選格工具，時段是商品類別，時格才是商品本身。

診所選中的同一個時格可以提供多位醫師使用。診所層時格不是庫存數量為 1 的商品，而是代表該診所在該分鐘有營業，並允許醫師在此基礎上設定自己的使用狀態。

### 醫師層

醫師可以理解成零售商。

醫師只能從診所已選用的時格中選擇自己的上班或服務時格。診所沒有選用的時格，醫師不能使用。

醫師選用後，該醫師的該分鐘時格成為醫師自己的零售商品。不同醫師可以使用同一個診所時格，但同一位醫師的同一個時格只能供一位消費者預約。

醫師設定暫停預約、休息、行政、內部保留、電話預約或現場掛號時，時格仍然存在，只是被標記為不同用途或不同販售狀態。

### 消費者層

消費者預約的是單一分鐘時格，不是一段自動延伸的服務時間。

例如消費者點選：

```text
09:00
```

代表購買該醫師在該日期的 09:00 這一個時格。

每幾分鐘顯示一次，例如每 5 分鐘、10 分鐘或 30 分鐘，是可販售時格的顯示與篩選間距，不代表消費者一次購買多個連續時格。

### 固定與臨時選擇器

固定模式與臨時模式都不是時格本身，而是選擇時格的工具。

固定模式是模式化選擇器，可以批量套用在每星期幾。例如每週一 09:00-12:00，代表每個星期一批量選擇 09:00 到 11:59 的分鐘時格。

臨時模式也是選擇器，但只套用在單一天，屬於單次使用。臨時模式的優先權高於固定模式，可以在指定日期臨時賣多一點、賣少一點或不賣。

臨時模式的套用流程：

1. 使用者在臨時模式完成時格設定。
2. 使用者點擊套用單日鈕。
3. 畫面出現日期選擇清單。
4. 使用者選擇要套用的日期。
5. 使用者點擊全部覆蓋鈕或局部覆蓋鈕。

套用方式：

- 全部覆蓋：該日期原本對應層級的時格資料全部覆寫為本次臨時設定結果。
- 局部覆蓋：使用加法覆寫，只新增或更新本次臨時設定的時格，其它未被本次選到的時格維持原狀。

其它時格來源與判斷規則，依本文後續診所固定模式、診所臨時模式、醫師固定模式與醫師臨時模式規則處理。

### 已售出時格

消費者預約成功後，該醫師的該分鐘時格成為已售出時格。

已售出時格不得因後續診所臨時停業、醫師停診或排班調整而被刪除。後續可以繼續操作，例如取消、改期、聯絡、完成或標記處理狀態，但必須保留預約紀錄。

任何時候只要時格已被消費者使用，就必須持續完成該使用流程應有的狀態，例如待報到、已報到、完成、過號、診所臨時取消或其它後續處理狀態。系統不得因上游診所或醫師規則變更，直接讓已被消費者使用的時格消失或中斷狀態流程。

若由消費者主動取消預約，預約紀錄仍需保留為取消紀錄；但該醫師時格可恢復該有的效力。恢復後是否可再次販售，需重新依當下診所有效時格、醫師有效時格、醫師販售狀態與其它預約限制判斷。

若診所臨時公休或臨時縮短營業，尚未售出的相關時格會變成不可販售或下架；已售出的時格仍保留，並繼續提供診所、醫師、使用狀態與統計所需資訊。

## 診所開業時間控制範圍

診所開業時間是系統可排班與可預約的最大時間範圍。

診所開業時間控制以下資料：

1. 診所的開業時間表
2. 可用的預約格
3. 醫師的上班時間表
4. 醫師上班時段是否開放預約

## 診所的開業時間表

診所開業時間表是診所自己對外營業的時間設定。

診所開業時間分為：

- 固定模式：每週固定開業時間。
- 臨時模式：單日臨時開業、臨時停業或縮短營業。

診所臨時模式優先於診所固定模式。

某一天的診所有效開業時間規則：

```text
如果該日期有診所臨時模式，使用診所臨時模式。
否則使用診所固定模式。
```

## 可用的預約格

一天最多有 1440 個 1 分鐘時格。

```text
24 小時 x 60 分鐘 = 1440 格
```

但只有落在診所有效開業時間內的時格，才可以變成可用的預約格。

診所開業時間決定當天可用預約格的最大範圍。

```text
全天 1440 格
> 診所有效開業時間
> 當天可用預約格上限
```

## 醫師的上班時間表

醫師的上班時間必須落在診所有效開業時間內。

這裡的「上班」指系統中的診所內服務時段，不討論診所外或非系統管理的工作。

醫師第一次建立上班時間表時，可以從診所開業時間建立第一版醫師固定上班表。

流程：

1. 診所先設定開業時間。
2. 醫師進入排班頁。
3. 若醫師尚未有上班時間表，系統可依診所固定開業時間產生可選時格。
4. 醫師選擇要使用的時段。
5. 系統建立第一版醫師固定上班時間表。

後續不論套用醫師固定模式或醫師臨時模式，都必須符合診所有效開業時間。

## 醫師上班時段的接診方式

醫師有上班，不代表每個上班時段都一定開放線上預約。

每個醫師上班時段可以設定不同接診方式，例如：

- 開放線上預約
- 暫停線上預約
- 只接受電話預約
- 現場掛號
- 內部保留時段
- 休息
- 行政
- 手術
- 檢查
- 指定病患或回診專用

因此醫師上班時間表不是單純的打勾表，而是醫師在不同時段的服務方式表。

## 時格狀態

一天 1440 格只是底層時間格。每一格可以在不同層級有不同狀態。

### 診所層狀態

- 未開業
- 開業
- 臨時開業
- 臨時停業

### 醫師層狀態

- 不上班
- 上班
- 臨時加班
- 臨時停診

### 預約層狀態

- 開放線上預約
- 電話預約
- 現場掛號
- 暫停預約
- 已被預約
- 內部保留

這樣可以避免把「醫師上班」直接等同於「可以線上預約」。

## 最終邏輯

核心規則：

```text
診所開業時間決定系統可排班與可預約的最大時間範圍。
醫師上班時間必須落在診所開業時間內。
醫師上班時段可以選擇不同接診方式。
只有接診方式為「開放線上預約」的時格，才會產生病患可選的線上預約格。
```

## 核心流程

1. 先設定診所開業時間

   使用診所固定模式或診所臨時模式，建立診所有效開業時間。

2. 再設定醫師上班時間

   醫師第一次設定時，可從診所開業時間建立醫師固定上班表。

3. 再設定醫師上班時段的接診方式

   例如線上預約、電話預約、現場掛號、內部保留、行政、檢查或手術。

4. 最後產生病患端可看到的預約格

   病患端只顯示同時符合以下條件的格子：

   - 診所有開
   - 醫師有上班
   - 該格接診方式為開放線上預約
   - 該格沒有被預約
   - 該格不是過去時間

## 模式矩陣規則

診所固定模式管控醫師固定模式，診所臨時模式管控醫師臨時模式。

### 醫師固定模式

醫師固定模式用來建立或更新醫師每週固定規則。

規則：

```text
時格來源：診所固定模式
套用目標：醫師固定規則
不可使用診所臨時模式時格
```

醫師固定模式只能抓取診所固定模式的時格。因為醫師固定模式會寫入每週固定規則，所以不能使用某一天診所臨時延長、臨時開業或臨時調整後的時格。

範例：

```text
診所固定星期三：09:00-12:00
診所臨時某星期三：09:00-18:00
```

醫師固定模式只能設定在：

```text
09:00-12:00
```

不可因為某一天診所臨時開到 18:00，就建立：

```text
醫師每週三 14:00-18:00
```

### 醫師臨時模式

醫師臨時模式用來建立或更新醫師單日臨時規則。

規則：

```text
時格來源：診所該日有效開業時間
套用目標：醫師臨時規則
若同步更新每週固定，必須重新符合診所固定模式
```

診所該日有效開業時間的判斷：

```text
如果該日有診所臨時模式，使用診所臨時模式。
否則使用診所固定模式。
```

因此醫師臨時模式可以使用診所臨時模式的時格。

範例：

```text
診所固定星期三：09:00-12:00
診所臨時某星期三：09:00-18:00
醫師臨時設定：14:00-18:00
```

此設定可以只套用今天，因為該日診所臨時開到 18:00。

但若要同步更新每週固定，必須重新檢查是否符合診所固定模式。上述 14:00-18:00 超出診所固定星期三 09:00-12:00，因此不能同步更新為每週固定規則。

若醫師臨時設定為：

```text
10:00-12:00
```

則可以同步更新每週固定，因為它也符合診所固定星期三 09:00-12:00。

### 矩陣摘要

```text
醫師固定模式
-> 時格來源：診所固定模式
-> 套用目標：醫師固定規則
-> 不可使用診所臨時模式時格

醫師臨時模式
-> 時格來源：診所該日有效開業時間
   如果該日有診所臨時模式，使用診所臨時模式
   否則使用診所固定模式
-> 套用目標：醫師臨時規則
-> 若同步更新每週固定，必須重新符合診所固定模式
```

一句話規則：

```text
臨時模式可以用診所臨時時格。
但只要要寫入每週固定，就必須重新通過診所固定模式驗證。
```

## 病患端可預約格判斷

病患端可看到的線上預約格必須符合：

```text
診所有效開業時間
∩ 醫師有效上班時間
∩ 接診方式為開放線上預約
∩ 尚未被預約
∩ 不是過去時間
```

若任一條件不成立，該格不顯示為可線上預約。
