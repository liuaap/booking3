# 個人卡片流程與規則

本文記錄「投稿形象建立」中的個人卡片規則。後續討論結果請持續補在本文件，作為實作與驗收依據。

## 名詞定義

- 個人卡片：醫師自行建立的投稿形象資料，也可稱為母卡。
- 診所收藏：診所收藏某張個人卡片，表示該診所希望使用此醫師形象資料。
- 診所收藏副本：同一張個人卡片被診所收藏後，在醫師端清單中產生的可獨立運作卡片。

## 個人卡片基本資料

個人卡片主要用於建立個人形象投稿，不包含診所或預約相關功能。

一個帳號可以建立多張個人卡片母卡。每張母卡代表一種獨立的投稿形象、執業身份或專長方向。

範例：

- 母卡 A：腸胃科 / 消化系疾病
- 母卡 B：睡眠醫學 / 失眠與睡眠障礙
- 母卡 C：長照醫療 / 中老年慢性病

多母卡規則：

- 同一個帳號可建立多張母卡。
- 每張母卡需要有自己的卡片 ID。
- 每張母卡可使用不同稱呼、科別、專長與個人ID。
- 每張母卡可獨立投入或退出邀請池。
- 每張母卡可獨立被診所搜尋、收藏或邀請。
- 每張母卡接受診所邀請後，各自產生診所收藏副本。
- 不同母卡產生的診所副本彼此獨立，不互相影響。
- UI 需讓使用者清楚知道目前操作的是哪一張母卡，避免誤以為是同一張卡重複建立。

卡片顯示欄位：

- 稱呼
- 科別
- 專長
- 個人ID

卡片操作：

- 投稿 / 收回
- 編輯形象
- 編輯個人
- 刪除卡片

個人卡片清單不顯示以下診所或預約功能：

- 上班時段
- 看診進度
- 編輯預約

## 醫師卡主流程

醫師卡流程暫定為：

1. 建立個人形象
2. 投入邀請池等待邀請
3. 被診所邀請
4. 開始上班

### 1. 建立個人形象

醫師先建立自己的個人卡片。此階段卡片只代表醫師的投稿形象，不屬於任何診所，也不提供預約、上班時段或看診進度功能。

此階段重點：

- 醫師填寫稱呼、科別、專長、個人ID。
- 醫師可編輯形象與個人資料。
- 卡片尚未進入診所端流程。
- 卡片尚不可被患者預約。

建議狀態名稱：

- 草稿
- 已建立

### 2. 投入邀請池等待邀請

醫師完成個人形象後，可選擇將卡片投入邀請池。投入邀請池後，診所可以搜尋或瀏覽此卡片，並向醫師發出邀請。

此階段重點：

- 醫師主動決定是否公開到邀請池。
- 母卡建立後預設為未投稿。
- 醫師端卡片需提供「投稿 / 收回」操作。
- 點選「投稿」後，該母卡進入邀請池。
- 點選「收回」後，該母卡退出邀請池，診所不可再從邀請池搜尋或發出新邀請。
- 診所可依稱呼、科別、專長、個人ID 搜尋。
- 卡片仍是個人卡片，不屬於任何診所。
- 卡片仍不可被患者預約。

卡片狀態顯示：

- 未投稿：僅醫師本人可見，不出現在邀請池。
- 已投稿：出現在邀請池，診所可搜尋與邀請。

收回限制暫定：

- 收回只影響未來曝光與新邀請。
- 已存在的診所邀請不會自動刪除，是否取消需另行討論。
- 已接受邀請產生的診所副本不受收回影響。

建議狀態名稱：

- 邀請池中
- 等待邀請

### 3. 被診所邀請

診所在邀請池中選擇醫師卡並發出邀請。醫師端需看到邀請來源與邀請狀態。

此階段重點：

- 一張個人卡片可同時收到多間診所邀請。
- 每間診所的邀請需要獨立顯示與處理。
- 醫師可接受或拒絕單一診所邀請。
- 尚未接受前，不建立上班時段與預約功能。

建議狀態名稱：

- 已被邀請
- 待回覆
- 已拒絕

### 4. 開始上班

醫師接受診所邀請後，該診所會取得一張診所收藏副本。此副本開始進入診所運作流程，可設定上班時段、預約時隔與看診進度。

此階段重點：

- 接受邀請後才建立或啟用診所收藏副本。
- 每一間診所對應一張可獨立運作的副本。
- 副本可顯示「已被某診所收藏」或「任職於某診所」。
- 副本開始具備診所、預約與上班相關功能。
- 不同診所副本的上班時段、預約設定與看診進度彼此獨立。

建議狀態名稱：

- 已接受邀請
- 已加入診所
- 上班中

## 流程狀態原則

個人卡片與診所副本應分成兩層狀態。

個人卡片狀態：

- 草稿
- 已建立
- 邀請池中
- 暫停邀請

診所邀請狀態：

- 待回覆
- 已接受
- 已拒絕
- 已取消

診所副本狀態：

- 尚未開始上班
- 上班中
- 暫停上班
- 已結束合作

原則：

- 個人卡片負責曝光與被邀請。
- 診所邀請負責醫師與單一診所之間的邀請流程。
- 診所副本負責接受邀請後的上班、預約與診所端運作。

## 診所收藏規則

當個人卡片被診所收藏時，醫師端卡片需要顯示收藏狀態。

顯示格式暫定：

- 已被「診所名稱」收藏

範例：

- 已被仁心健康診所收藏
- 已被康博診所收藏

## 多診所收藏規則

若同一張個人卡片被 2 間以上診所收藏，系統需自動拆分顯示為多張卡片。

規則：

- 每一間收藏診所對應一張診所收藏副本。
- 每張副本顯示相同的個人基礎資料。
- 每張副本顯示不同的收藏診所名稱。
- 每張副本可獨立運作。

範例：

同一張個人卡片被 A 診所與 B 診所收藏時，醫師端清單顯示：

- 卡片 1：稱呼、科別、專長、個人ID，已被 A 診所收藏
- 卡片 2：稱呼、科別、專長、個人ID，已被 B 診所收藏

## 獨立運作原則

診所收藏副本不是單純的畫面重複顯示，後續應可各自維護與操作。

目前確認方向：

- 母卡負責保存醫師原始投稿形象。
- 診所收藏副本可保留來源母卡關聯。
- 不同診所收藏副本可以有不同收藏診所名稱。
- 不同診所收藏副本後續可擴充為各自獨立設定。

待討論：

- 副本是否允許獨立修改稱呼、科別、專長、個人ID。
- 副本修改後是否回寫母卡。
- 母卡更新後是否同步更新所有副本。
- 診所取消收藏後，副本是否刪除、隱藏或保留歷史。
- 醫師刪除母卡時，已收藏副本如何處理。

已確認規則：

- 副本允許獨立修改稱呼、科別、專長與個人ID。
- 副本修改後不需要回寫母卡。
- 母卡更新後，可建立同步機制，由持卡人選擇是否同步到哪些副卡、同步哪些欄位。
- 醫師刪除母卡時，已存在的診所副本屬於獨立事件，不受母卡刪除影響。
- 診所取消收藏或刪除副本時，醫師需要可以將該副本資產帶走。

### 副本被診所刪除時的帶走規則

當診所刪除、取消收藏或結束合作副本時，該副本不得直接讓醫師累積的形象資產消失。系統需提供「帶走副本」或「轉存為新母卡」的能力。

可帶走內容：

- 稱呼
- 科別
- 專長
- 個人ID 或重新產生的新個人ID
- 照片
- 個人形象內容
- 卡片說明 / 自我介紹
- 診所副本中累積的醫師個人評論摘要
- 可判定為針對醫師本人的評論
- 來源副本紀錄
- 原合作診所名稱
- 原副本建立時間
- 原副本結束時間

不帶走內容：

- 上班時段
- 預約時隔
- 看診進度
- 診間 / 房間設定
- 診所內部備註
- 患者預約資料
- 診所權限設定
- 診所專屬通知紀錄

評論帶走原則：

- 可帶走與醫師本人服務品質相關的評論或評論摘要。
- 涉及診所環境、櫃台服務、等候時間、設備、收費、地點等內容，不應直接轉成母卡評論。
- 轉存後的評論需標示來源診所與來源副本。
- 初期建議優先帶走評論摘要，原文評論是否公開需另行討論。

工程期程：

- 此功能列入下一期工程項目。
- 本期不實作副本帶走、評論轉存或轉存為新母卡功能。

## 初步資料關聯

建議資料概念：

- 母卡 ID：個人卡片本身的唯一識別。
- 個人ID：醫師對外顯示或搜尋使用的 ID。
- 收藏診所 ID：收藏此卡片的診所識別。
- 收藏診所名稱：顯示於卡片上的收藏狀態。
- 來源母卡 ID：診所收藏副本對應的原始個人卡片。

## 醫師排班設定頁規則

醫師排班頁可以在第一次沒有醫師上班表時，從診所營業時間建立第一版醫師固定上班時間。建立後，排班頁再依醫師上班時間表產生可預約時隔。

### 基本流程

1. 診所先設定營業時段。
2. 醫師進入排班頁。
3. 若醫師尚未有上班時間表，排班頁可切換到診所營業時間，從診所營業時段建立第一版醫師固定上班時間。
4. 若醫師已有上班時間表，系統以「診所營業時段」與「醫師上班時段」取交集，得到該醫師可排班的有效時段。
5. 排班頁選擇星期幾或日期後，系統讀取該日有效時段。
6. 系統依有效時段自動產生時段卡。
7. 每張時段卡再依「每幾分鐘」產生可預約時隔。
8. 醫師可在時段卡內開啟、暫停或全選該卡片內的時隔。

### 星期與時段卡規則

- 排班頁必須先選星期幾或日期，再顯示對應的時段卡。
- 時段卡由該星期的醫師上班時段自動產生。
- 一個上班時段對應一張時段卡。
- 如果星期一有三個上班時段，排班頁就顯示三張時段卡。
- 每張時段卡的起訖時間必須與該上班時段相符，或落在該上班時段內。
- 時段卡不可超出該星期的醫師上班時段。
- 若該星期沒有上班時段，則不顯示時段卡，也不產生可預約時隔。

範例：

- 星期一上班時段為 09:00~09:30、14:00~14:30、18:00~21:00。
- 排班頁選擇星期一時，系統自動顯示三張時段卡。
- 時段卡 1：09:00~09:30。
- 時段卡 2：14:00~14:30。
- 時段卡 3：18:00~21:00。

### 時隔產生規則

- 「每幾分鐘」不是決定可用範圍，而是在有效時段確認後，用來切出預約時隔。
- 有效範圍需同時符合：使用者選擇的星期或日期、診所營業時段、醫師上班時段。
- 排班頁若仍保留時間選擇器，即使選擇 00:00~24:00，也只能顯示符合有效時段的時隔。
- 超出醫師上班時段或診所營業時段的時隔，一律不顯示、不儲存。
- 每張時段卡可獨立設定時隔，例如每 1 分鐘、5 分鐘、10 分鐘或 30 分鐘。
- 全選、全部暫停或單一時隔開關，只作用於該張時段卡。

範例：

- 醫師星期三有效上班時段為 14:00~14:30。
- 時隔設定為每 1 分鐘時，顯示 14:00、14:01、14:02，直到 14:29。
- 時隔設定為每 30 分鐘時，只顯示 14:00。
- 不應顯示 14:30 之後的時間，也不應顯示其他非上班時段。

### 固定模式與臨時模式

- 固定模式：依選定星期的時段卡設定，更新該醫師每週固定可預約時隔。
- 臨時模式：預設套用單一日期，用於單日加診、停診或單日特殊調整；也可選擇同步更新每週固定規則。
- 固定模式與臨時模式都必須遵守同一套有效時段比對規則。
- 臨時模式不可因為是單日設定，就產生超出醫師上班時段或診所營業時段的時隔。

## 待補充討論

後續需補上的流程：

- 診所如何收藏個人卡片。
- 醫師端如何收到收藏通知。
- 醫師是否需要同意診所收藏。
- 診所收藏後是否立即產生副本。
- 副本進入診所端後的可操作功能。
- 收藏副本與診所預約功能何時產生關聯。
- 多診所收藏時的排序與顯示規則。
