# 雙軌制時格實作規則

本文記錄固定模式與臨時模式的雙軌制時格操作方法，作為後續排班頁與預約輸出實作依據。

## 核心概念

系統每天都有 1440 個 1 分鐘基礎時格。

每一個診所、每一天，系統需要準備兩份時格軌道：

1. 固定軌 fixed track
2. 臨時軌 temporary track

固定軌只給固定模式使用。

臨時軌只給臨時模式使用。

兩條軌道各自運作、各自儲存、各自輸出結果。需要產生病患可預約結果時，才依規則交叉比對或選擇輸出。

## 資料結構

每一天至少要能表示：

```js
{
  fixed: {
    trackType: "fixed",
    minutes: [/* 1440 個分鐘時格 */]
  },
  temporary: {
    trackType: "temporary",
    minutes: [/* 1440 個分鐘時格 */]
  }
}
```

每一個分鐘時格至少包含：

```js
{
  minute: 1260,
  time: "21:00",
  enabled: true,
  source: "fixed" | "temporary",
  booked: false
}
```

可以先用時段區間或 enabledTimes 實作，但讀取時必須轉成固定軌與臨時軌的概念。

## 固定軌

固定軌代表每週固定規則推算出來的時格。

固定軌來源：

- 診所固定營業時間
- 醫師固定上班規則
- 醫師固定時格狀態

固定軌不可使用：

- 診所臨時加開時段
- 診所臨時減班時段
- 診所單日臨時營業時間
- 醫師單日臨時調整

例如：

診所星期五固定營業：

```text
09:00-12:00
14:00-17:00
18:30-20:30
```

即使 2026-05-29 臨時加開：

```text
21:00-23:00
```

固定套用星期五仍只能產生：

```text
09:00-12:00
14:00-17:00
18:30-20:30
```

不得產生：

```text
21:00-23:00
```

## 臨時軌

臨時軌代表單日臨時規則推算出來的時格。

臨時軌來源：

- 該日期診所有效營業時間
- 該日期診所臨時加開
- 該日期診所臨時減班或休診
- 醫師該日期臨時調整
- 醫師該日期時格狀態

臨時模式優先使用該日期的診所有效時段。

判斷順序：

1. 如果該日期有診所臨時規則，使用診所臨時規則。
2. 如果該日期沒有診所臨時規則，使用診所固定營業時間。
3. 醫師在該日期編輯的結果，寫入臨時軌。

例如：

2026-05-29 診所臨時加開：

```text
21:00-23:00
```

臨時套用 2026-05-29 時，可以產生：

```text
21:00-23:00
```

並應以加開 icon 或加開狀態標示。

## 操作流程

### 固定套用

固定套用流程：

1. 使用者選擇固定套用。
2. 使用者選擇星期幾。
3. 系統讀取該星期的固定軌來源。
4. 下方依固定軌來源產生時段卡。
5. 使用者編輯醫師自己的時格狀態。
6. 使用者按儲存編輯。
7. 系統詢問是 / 否。
8. 確認後寫入醫師 fixed track。

固定套用只能寫入 fixed track。

固定套用不得寫入 temporary track。

固定套用不得讀取或混入臨時加開時段。

### 臨時套用

臨時套用流程：

1. 使用者選擇臨時套用。
2. 使用者選擇日期。
3. 系統讀取該日期的 temporary track 來源。
4. 若該日沒有臨時規則，臨時軌可由診所固定營業時間初始化。
5. 下方依該日期有效時段產生時段卡。
6. 使用者編輯醫師自己的時格狀態。
7. 使用者按儲存編輯。
8. 系統詢問是 / 否。
9. 確認後寫入醫師 temporary track。

臨時套用只能寫入 temporary track。

臨時套用不得直接改寫 fixed track。

## 輸出規則

對病患輸出可預約時格時，系統不得直接混用兩條軌道。

建議輸出順序：

1. 如果該日期有 temporary track 規則，優先輸出 temporary track。
2. 如果該日期沒有 temporary track 規則，輸出 fixed track。
3. 已被預約或已售出的時格必須保留狀態，不得因軌道切換而消失。

簡化範例：

```js
const effectiveTrack = hasTemporaryRule(date)
  ? schedule[clinicId][date].temporary
  : schedule[clinicId][date].fixed;
```

如果未來需要更細的交叉比對，必須明確定義輸入與輸出，例如：

```js
const bookableSlots = intersect(
  clinicEffectiveTrack,
  doctorEffectiveTrack,
  bookingState
);
```

## UI 標示規則

固定時段：

- 使用一般固定時段 icon。
- 不標示加開或減班。

臨時加開：

- 使用加開 icon。
- 建議使用 `+` 或綠色標示。
- 只出現在臨時軌或該日期有效時段參考中。

臨時減班或休診：

- 使用減班 icon。
- 建議使用 `-` 或橘色標示。
- 只影響臨時軌。

## 禁止事項

- 固定模式不得讀取臨時加開時段。
- 固定模式不得因某一天臨時營業而更新每週固定規則。
- 臨時模式不得直接覆寫固定軌。
- 只隱藏 UI 按鈕但保留舊流程，不能視為完成規則調整。
- 不得用同一份 selectedDate 同時代表固定星期與臨時日期而不做區分。

## 實作檢查清單

- 固定套用星期五時，只顯示診所星期五固定時段。
- 臨時套用 2026-05-29 時，可顯示 2026-05-29 的臨時加開時段。
- 固定週表不顯示臨時加開時段。
- 臨時加開時段有不同 icon。
- 儲存固定模式時，只更新 fixed track。
- 儲存臨時模式時，只更新 temporary track。
- 病患預約列表優先使用 temporary track；沒有 temporary track 才使用 fixed track。
- 已預約時格不因固定軌或臨時軌重算而消失。
